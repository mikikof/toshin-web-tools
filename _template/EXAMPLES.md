## 既存ツールから「真似する/しない」(重要、量産時の指針)

新規ツールは雛形(`_template/saas-{offline,canvas,d3,three}.html`)を起点にする。
既存 32 ツールは参考にするが、**そのまま真似してはいけない部分** がある。

### 全既存ツールから「真似する」もの(共通パターン)

- ✅ **教育構造**: プリセット 3 つ → スライダー → 比較 / 反例 の段階配置
- ✅ **状態管理アプローチ**: `state` オブジェクトを集中管理してリアルタイム再描画
- ✅ **可視化の選び方**: テーマに応じた SVG / Canvas / D3 / Three の使い分け
- ✅ **解説の長さ**: 教師目線で長文化せず、触らせる UI を主役に
- ✅ MathML + code 併記(オフライン優先のとき)
- ✅ プリセットボタンの「シナリオ命名」(「SGD でジグザグを体験 ➜」のような行為文)

### 既存ツールから「真似しない」もの(雛形と齟齬)

- ❌ **Tailwind CDN**(多くの既存ツールが使用) → 共通 `tool-style.css` で代替
- ❌ **React UMD**(4-3, linear/2-2) → Vanilla で書く(ファイルサイズ・可読性)
- ❌ **inline style 大量**(各 `style="..."` 直書き) → `tool-` prefix の class を使う
- ❌ **独自カラーパレット**(各ツールごとに `:root` で別の色定義) → 共通 `tool-style.css` のトークンに統一
- ❌ **CDN 多用**(jQuery, Bootstrap 等) → 必要な 1〜2 ライブラリだけに絞る
- ❌ **ハードコードされた絵文字 / 過剰装飾**(💪🎉 等の装飾) → 学習の主役を妨げない範囲で

### 既存ツール参照の手順(Step 2 で使う)

```
1. EXAMPLES.md の表でテーマ近似ツールを検索(例: 「テイラー → 5-2」)
2. tools/calculus/<該当>.html を Read
3. 「教育構造」「状態管理」「プリセット文言」を抽出
4. それ以外(CSS / Tailwind / React)は **無視して** 雛形に戻る
5. 雛形の TODO コメントを「抽出した教育設計」で埋める
```

## 自動チェック支援(SKILL Step 7 用)

新規ツール完成時、以下の grep で機械的に確認できる:

```bash
FILE="tools/calculus/<chapter>-<section>.html"

# TODO 残存検出(雛形の TODO コメントが埋まっているか)
grep -c "TODO" "$FILE"

# CDN 依存の検出(完全オフライン要件のとき)
grep -E '(<script[^>]*src="https://|<link[^>]*href="https://)' "$FILE"

# MathML + code 併記の確認(両方あるか)
grep -c "<math" "$FILE"
grep -c "tool-mono" "$FILE"

# label for 属性の確認(A11y)
grep -E '<label for="' "$FILE" | wc -l

# aria-live の確認(KPI 等)
grep "aria-live" "$FILE"
```

これらを SKILL.md Step 7 で実行する。
