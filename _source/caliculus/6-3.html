                    <h3 class="text-xl font-bold mb-2 text-indigo-600">1. ディリクレ関数：リーマン積分を壊す「意地悪」な例</h3>

                    <p>ルベーグ積分のすごさを示す典型例が「ディリクレ関数」です。これは「xが有理数なら1、無理数なら0」という値を取る関数で、無数の不連続点を持ちます。</p>

                    <ul class="list-disc list-inside mt-2 space-y-1 pl-4">

                        <li><b>リーマン積分の問題点:</b> どんなに細い短冊を作っても、その中には必ず有理数と無理数の両方が含まれます。そのため、短冊の高さを「1」と取ることも「0」と取ることもできてしまい、面積の合計値は決して一つの値に収束しません。計算が破綻してしまうのです。</li>

                        <li><b>ルベーグ積分の鮮やかな解決策:</b> ルベーグはx軸ではなくy軸を見ます。「値が1になる点の集まり（有理数全体）」と「値が0になる点の集まり（無理数全体）」に分け、それぞれの集合の「大きさ（測度）」を考えます。数直線上では、有理数はスカスカで測度は0、無理数はびっしりで測度は1（区間[0,1]の場合）となります。よって面積は <code class="bg-gray-100 p-1 rounded">(値 1 × 測度 0) + (値 0 × 測度 1) = 0</code> と、いとも簡単に確定します。</li>

                    </ul>

                    <p>「サイコロの期待値」を計算するとき、私たちは自然にルベーグ積分的な考え方をしています。計算式は `(1の目 × 1/6) + (2の目 × 1/6) + ...` となりますが、これは「出る目（<b>値</b>）」ごとにグループ分けし、「その値が出る<b>確率（測度）</b>」を掛けて足し合わせる、まさしくルベーグの発想です。</p>

                    <p class="mt-2">宝くじのように、確率が連続的な部分（ハズレ）と離散的な部分（アタリ）で混在している場合、リーマン積分ではうまく扱えません。しかし、ルベーグ積分なら、出る値（賞金）でグループ分けし、それぞれの確率（測度）を掛けて足し合わせるだけで、期待値を完璧に計算できます。</p>

                setupCanvas() {

                    this.dpr = window.devicePixelRatio || 1;

                    const rect = this.canvas.parentElement.getBoundingClientRect();

                    this.width = rect.width;

                    this.height = rect.height;

                    this.canvas.width = this.width * this.dpr;

                    this.canvas.height = this.height * this.dpr;

                    this.canvas.style.width = `${this.width}px`;

                    this.canvas.style.height = `${this.height}px`;

                    this.ctx.scale(this.dpr, this.dpr);

                    this.drawScene();

                }


                toCX(x) { return this.graph.padding + (x - this.graph.xMin) / (this.graph.xMax - this.graph.xMin) * (this.width - 2 * this.graph.padding); }

                toCY(y) { return this.height - this.graph.padding - (y - this.graph.yMin) / (this.graph.yMax - this.graph.yMin) * (this.height - 2 * this.graph.padding); }


                drawAxesAndFunction() {

                    this.ctx.clearRect(0, 0, this.width, this.height);

                    this.ctx.strokeStyle = '#d1d5db';

                    this.ctx.lineWidth = 1;

                    this.ctx.beginPath();

                    this.ctx.moveTo(this.graph.padding, this.toCY(0));

                    this.ctx.lineTo(this.width - this.graph.padding, this.toCY(0));

                    this.ctx.moveTo(this.toCX(this.graph.xMin), this.graph.padding);

                    this.ctx.lineTo(this.toCX(this.graph.xMin), this.height - this.graph.padding);

                    this.ctx.stroke();

                    this.ctx.lineWidth = 3;

                    this.ctx.beginPath();

                        this.ctx.lineTo(this.toCX(x), this.toCY(this.currentFunc(x)));

                    }

                    this.ctx.stroke();

                }


                drawRiemann() {

                    const dx = (this.graph.xMax - this.graph.xMin) / this.partitions;

                    let totalArea = 0;


                    for (let i = 0; i < this.partitions; i++) {

                        const xStart = this.graph.xMin + i * dx;

                        const y = this.currentFunc(xStart);

                        

                        totalArea += y * dx;

                    }

                    this.areaLabel.textContent = totalArea.toFixed(2);

                }


                drawLebesgue() {

                    const yMinIntegration = 0;

                    const yMaxIntegration = this.graph.yMax;

                    const dy = (yMaxIntegration - yMinIntegration) / this.partitions;

                    let totalArea = 0;


                    for (let i = 0; i < this.partitions; i++) {

                        const y = yMinIntegration + i * dy;

                        

                        let measure = 0;

                        let inSet = false;

                        for (let x = this.graph.xMin; x <= this.graph.xMax; x += sampleStep) {

                            if (currentlyInSet && !inSet) startX = x;

                            if (!currentlyInSet && inSet) measure += (x - startX);

                            inSet = currentlyInSet;

                        }

                        if (inSet) measure += (this.graph.xMax - startX);


                        if (measure > 0) {

                            totalArea += dy * measure;

                            const hue = (y / this.graph.yMax) * 240 + 120;

                            this.ctx.fillStyle = `hsla(${hue}, 90%, 70%, 0.7)`;

                            this.ctx.strokeStyle = `hsla(${hue}, 90%, 40%, 1)`;

                            this.ctx.lineWidth = 0.5;


                            inSet = false;

                            startX = 0;

                            for (let x = this.graph.xMin; x <= this.graph.xMax; x += sampleStep) {

                                 const fx = this.currentFunc(x);

                                 const currentlyInSet = (fx >= y);

                                 if (currentlyInSet && !inSet) startX = x;

                                 if (!currentlyInSet && inSet) {

                                     this.ctx.fillRect(this.toCX(startX), this.toCY(y + dy), this.toCX(x) - this.toCX(startX), this.toCY(y) - this.toCY(y + dy));

                                     this.ctx.strokeRect(this.toCX(startX), this.toCY(y + dy), this.toCX(x) - this.toCX(startX), this.toCY(y) - this.toCY(y + dy));

                                 }

                                 inSet = currentlyInSet;

                            }

                            if(inSet){

                            }

                        }

                    }

                    this.areaLabel.textContent = totalArea.toFixed(2);

                }

        <h2>8. コーシー・シュワルツの不等式</h2>

        <div class="formula">$$ \left( \int_{\alpha}^{\beta} f(t)g(t) dt \right)^2 \le \left( \int_{\alpha}^{\beta} f(t)^2 dt \right) \left( \int_{\alpha}^{\beta} g(t)^2 dt \right) $$</div>

        <div class="explanation" style="background-color: #e8f6f3; border-left-color: #1abc9c;">

            <p><strong>💡 なぜこうなるの？ベクトルで考えてみよう！</strong></p>

            この不等式をグラフで直接イメージするのは難しいですが、高校で習うベクトルの「内積」で考えると、とてもスッキリします。<br>

            2つのベクトル $\vec{a}$ と $\vec{b}$ があるとき、内積に関する不等式は $(\vec{a} \cdot \vec{b})^2 \le |\vec{a}|^2 |\vec{b}|^2$ でした。（等号は2つのベクトルが平行なときに成り立ちます）

            <p style="margin-top:10px;">実は、関数も一種の「無限次元のベクトル」と見なすことができます。その世界では、<br>

            - 関数の<b>内積</b>が $\int_{\alpha}^{\beta} f(t)g(t)dt$<br>

            - 関数の<b>長さの2乗</b>が $\int_{\alpha}^{\beta} f(t)^2 dt$<br>

            に対応します。こう考えると、コーシー・シュワルツの不等式は、ベクトルの内積の性質を関数に拡張したもの、と直感的に理解できます。</p>

/**

 * 数値積分（リーマン和による近似）

 * @param {function} func - 積分対象の関数

 * @param {number} a - 積分区間の下端

 * @param {number} b - 積分区間の上端

 * @param {number} n - 分割数

 * @returns {number} 積分結果の近似値

 */

function integrate(func, a, b, n = 200) {

    if (a === b) return 0;

    const dx = (b - a) / n;

    let sum = 0;

    for (let i = 0; i < n; i++) {

        sum += func(a + i * dx);

    }

    return sum * dx;

}

    function update6() {

        chart6.data.datasets[2].data = chartLabels.map((t) => (t >= alpha && t <= beta) ? avg_height : null);
